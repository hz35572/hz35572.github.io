OpenCode：终端中的 AI 编程利器——从安装到精通

前言

在 AI 辅助编程的浪潮中，你可能已经用过 Cursor、Copilot Chat 等 GUI 工具，但如果你是一个终端重度用户，希望在命令行中拥有一位”智能搭档”，那么 OpenCode 绝对值得你关注。

OpenCode 是一款开源的终端 AI 编程代理，基于 Go 语言构建，使用 Bubble Tea 框架打造了流畅的 TUI（终端用户界面）。它支持 OpenAI、Anthropic Claude、Google Gemini、AWS Bedrock、Groq、Azure OpenAI 等 75+ 种 LLM 提供商，让你在终端中就能享受到强大的 AI 编程辅助。

⚠️ 更新提示：OpenCode 项目现已更名为 Crush，由原开发者与 Charm 团队共同维护。本文内容基于 OpenCode 版本撰写，核心概念与操作方式在新项目中保持一致。

一、OpenCode 是什么？

简单来说，OpenCode 就是你的终端里的 AI 编程助手。它不仅能回答问题，还能直接操作你的代码——执行命令、搜索文件、修改代码，全部在终端中完成。

核心特性一览

特性 说明
交互式 TUI 基于 Bubble Tea 构建的终端界面，流畅美观
多模型支持 支持 75+ 种 LLM 模型，包括 Claude、GPT、Gemini 等
会话管理 支持多会话并行，方便切换上下文
工具集成 AI 可执行命令、搜索文件、修改代码
Vim 风格编辑器 内置编辑器，支持快捷键操作
LSP 集成 支持语言服务器协议，提供代码智能感知
文件变更追踪 可视化追踪会话中的文件修改
自定义命令 支持创建自定义 Markdown 命令模板
MCP 协议 通过 Model Context Protocol 扩展外部工具
隐私优先 不存储你的代码和上下文数据

Q4：上下文丢失

问题：长时间对话后 AI “忘记”了前面的内容。

解决：

  1. 确保开启 autoCompact："autoCompact": true
  2. 手动压缩会话：按 Ctrl+K → 选择 Compact Session
  3. 适时创建新会话：按 Ctrl+N

Q5：文件修改权限被拒绝

问题：AI 尝试修改文件时弹出权限确认。

解决：

  • 按 a 允许本次操作
  • 按 A 允许整个会话的操作
  • 在非交互模式下所有权限自动批准

Q6：Windows 下的特殊问题

问题：Windows 终端显示异常或快捷键不生效。

  1. 推荐使用 Windows Terminal 或 WezTerm

八、使用技巧总结

  1. 善用项目级配置：在项目根目录创建 .opencode.json，为不同项目设置不同模型和参数
  2. 创建自定义命令：将常用的审查、部署、测试流程封装成命令，一键执行
  3. 利用多会话：同时开启多个会话处理不同任务，互不干扰
  4. 巧用非交互模式：在 CI/CD 流水线中使用 opencode -p 实现自动化代码审查
  5. MCP 扩展生态：接入数据库、搜索引擎等 MCP 服务器，让 AI 获取更多上下文
  6. 定期压缩会话：长对话时注意 token 消耗，适时压缩或创建新会话
  7. 分享会话链接：OpenCode 支持生成会话分享链接，方便团队协作和问题排查
